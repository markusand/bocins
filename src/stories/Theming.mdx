# Theming

Bocins is styled entirely through CSS custom properties — no preprocessors, no overrides, no `!important`. Drop in your design tokens and components adapt automatically.

Each component exposes two levels of customization:

- **Theme var** (`--{component}-*`) — set on `:root` to style all instances globally
- **Instance var** (shorter internal name) — set on a specific element to override just that one

```css
/* Theme all buttons */
:root {
  --btn-color: royalblue;
}

/* Override a specific button */
.btn-danger {
  --color: crimson;
}
```

A set of **global design tokens** flow through every component. Set them once and the entire UI responds:

| Token              | Used for                   |
| ------------------ | -------------------------- |
| `--bg-color`       | Panel / input background   |
| `--text-color`     | Text color                 |
| `--border-color`   | Input / panel border       |
| `--border-width`   | Input / panel border width |
| `--radius`         | Border radius fallback     |
| `--accent-color`   | Focus / active highlight   |
| `--disabled-color` | Disabled state             |
| `--error-color`    | Invalid / error state      |

Components that respond to these tokens are noted in each section below.
